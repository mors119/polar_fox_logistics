# 기여 가이드

## 기본 원칙

- 이 저장소는 애플리케이션입니다. 프레임워크처럼 설계하지 않습니다.
- 추상화보다 읽기 쉬운 직접 코드를 우선합니다.
- 구조보다 생산성이 중요할 때는 생산성을 선택합니다.
- 기능 추가 시 테스트와 문서를 함께 갱신합니다.

## 코드 배치 기준

- 기능 로직은 기능 폴더 안에 둡니다.
- 공용 유틸리티만 `src/shared`로 올립니다.
- 진입점은 얇게 두고 실제 동작은 기능 함수가 수행하게 합니다.
- Apps Script API 호출은 필요한 작은 helper로 감싸되, wrapper 위에 wrapper를 만들지 않습니다.

## Pull Request 체크리스트

- `npm run format`
- `npm run lint`
- `npm run typecheck`
- `npm run test`
- `npm run build`

## 리뷰 기준

- 파일 위치가 기능 기준으로 자연스러운지
- 추상화가 실제 이득보다 인지 부담을 키우지 않는지
- 설정값이 하드코딩되지 않았는지
- 시트/Drive/Gmail 쓰기 동작이 의도적으로 보이는지
- 테스트가 실제 동작을 설명하는지
